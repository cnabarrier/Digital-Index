\## Core rule

Avoid generic “AI slop” aesthetics. The UI must look like a human designer made deliberate choices.



\## Design requirements

\- Typography: pick interesting fonts. Avoid generic fonts (Arial, Inter, Roboto, system defaults).

\- Color \& theme: commit to a cohesive aesthetic using CSS variables. Strong dominant colors plus sharp accents.

\- Motion: use a small number of high-impact animations (staggered entrance on load beats random micro-animations).

\- Backgrounds: add depth (layered gradients, subtle patterns). Avoid flat default backgrounds.



\## Avoid

\- Overused font families (Inter, Roboto, Arial, system fonts)

\- Cliché purple-on-white gradients

\- Cookie-cutter component layouts

\- “Pretty but generic” outputs



\## What “good” looks like

\- One clear design direction

\- Consistent spacing system

\- Deliberate typography hierarchy

\- A few polished interactions that feel intentional



\## Operating constraints

\- Keep changes minimal and localized.

\- Do not rewrite unrelated HTML/JS while doing design work.
